删除案例模板

案例模板允许预填案例对象，如任务、标签和备注.
它可用于添加程序，定义如何应对特定类型的事件（网络钓鱼、勒索软件、APT......）.

案例模板可以 JSON 文件形式上传，也可以在添加/编辑案例模板用户界面中添加和编辑.
案例模板分类
案例模板的定义如下. 

{
    "name": "ransomware_infection",
    "display_name": "勒索病毒感染模板",
    "description": "本案例模板描述了处理受到勒索软件攻击的信息系统的第一反应任务.",
    "author": "DFIR-IRIS",
    "title_prefix": "RANS",
    "summary": "# Context \n\n\n # Contact \n\n\n # Actions \n\n\n",
    "tags": ["ransomware","malware"],
    "tasks": [
        {
            "title": "识别边界",
            "description": "必须确定入侵范围",
            "tags": ["identify"]
        },
        {
            "title": "收集被入侵的主机",
            "description": "部署Velociraptor并开始收集证据",
            "tags": ["collect", "velociraptor"]
        },
        {
            "title": "包含"
        }
    ],
    "note_directories": [
        {
            "title": "识别",
            "notes": [
                {
                    "title": "识别被入侵的账户",
                    "content": "# 观察\n\n"
                }
            ]
        },
        {
            "title": "收集",
            "notes": [
                {
                    "title": "Velociraptor部署"
                },
                {
                    "title": "资产已收集",
                    "content": "# 资产已收集\n\n# 资产未收集"
                }
            ]
        }
    ]
}

字段类型

支持的字段类型有:
  • name: 案例模板名称（必填）.
  • display_name: 案例模板的显示名称.
  • description: 案例模板说明.
  • author: 案例模板的作者（与当前用户无关）.
  • title_prefix: 为案例标题添加前缀.
  • summary: 填充摘要的内容.
  • tags: 案例标签列表.
  • tasks: 定义任务的字典列表。任务由标题（必填）、描述和标记列表定义。
  • note_groups: 定义备注组的词典列表。备注组由标题（必填）和注释列表定义。备注有标题（必填）和内容
