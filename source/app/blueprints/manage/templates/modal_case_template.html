案例模板允许预填案例对象，如任务、标签和备注.
它可用于添加程序，定义如何应对特定类型的事件（网络钓鱼、勒索软件、APT...）

案例模板可以在此用户界面中添加和编辑，也可以作为 JSON 文件上传.

IRIS 将尝试在提交之前验证案例模板的内容.

案例模板分类

字段类型

支持的字段类型有:
  • name: 案例模板名称（必填）.
  • display_name: 案例模板的显示名称.
  • description: 案例模板描述.
  • author: 案例模板的作者（与当前用户无关）.
  • classification: 案例模板的分类。应为小写名称，与 IRIS 现有分类相匹配.
  • title_prefix: 为案例标题添加前缀.
  • summary: 使用内容来预填摘要。
  • tags: 案例标签列表.
  • tasks: 定义任务的字典列表。任务由标题（必填）、描述和标记列表定义.
  • note_directories: 定义注释目录的词典列表。注释目录由标题（必填）和注释列表定义。注释有标题（必填）和内容
案例模板示例
案例模板的定义如下. 

{
    "name": "ransomware_infection",
    "display_name": "勒索软件感染模板",
    "description": "本案例模板介绍了处理受勒索软件入侵的信息系统的第一反应任务.",
    "author": "DFIR-IRIS",
    "classification": "恶意代码:勒索软件",
    "title_prefix": "[RANS]",
    "summary": "# 内容 \n\n\n# 联系方式 \n\n\n# 动作 \n\n\n",
    "tags": ["勒索软件","恶意软件"],
    "tasks": [
        {
            "title": "识别范围",
            "description": "必须确定入侵范围",
            "tags": ["识别"]
        },
        {
            "title": "收集被入侵主机",
            "description": "部署Velociraptor并开始收集证据",
            "tags": ["收集", "velociraptor"]
        },
        {
            "title": "遏制"
        }
    ],
    "note_directories": [
        {
            "title": "识别",
            "notes": [
                {
                    "title": "识别被入侵的账户",
                    "content": "# 观察\n\n"
                }
            ]
        },
        {
            "title": "收集",
            "notes": [
                {
                    "title": "Velociraptor部署"
                },
                {
                    "title": "资产已收集",
                    "content": "# 资产已收集\n\n# 资产未收集"
                }
            ]
        }
    ]
}
